Front End Development Code Review Process

Created by Kalyan Sripathi on Jun 15, 2018 Go to start of metadata

Purpose of the front end development

I want to let the team know that, this is in no way supposed to be personal or a critique of your technical competence. We all have our areas of strengths and areas we can improve on. This is a chance to give the team and also the reviewer a chance to improve their technical skills and also get a fresh perspective on things.

How is the reviewer selected?


The reviewer is selected based on their general experience with front-end development, functional and domain knowledge. If someone has specific experience with the project or the specific technical requirements they will be given preference.We will be selecting reviewers for different projects from India and NA , so if you think a certain project/programming language is your area of expertise please reach out to us.

âœ¡What will be reviewed?

Here is a blueprint for the review process. Obviously, as we do more of these and learn, we will refine the process and template of the review process. But for now, here is what we think are important factors to assess.

Framework decisions: Are correct framework decisions made? For example, Is the app built in Angular, REACT etc? If so why choose one over the other? If no proper front-end-framework is used (eg just using JQUERY or plain javascript), why did we make the decision? (Backend product constraints, developers aren't aware of modern frameworks etc) Node modules/External libraries: What node modules/libraries are being used? Does that follow our choice of node modules in other projects? Do we need them or is writing our own solution or using another alternative more advisable?

Code Quality : The overall quality of the code. Eg: Using vars where consts and lets should be used. Defining the proper interfaces for objects, using the correct types, passing arguments correctly, not looping through things unnecessarily.

Modularization: Defining the right modules, components,nested components, classes and functions. Grouping all services properly, folder structure.
